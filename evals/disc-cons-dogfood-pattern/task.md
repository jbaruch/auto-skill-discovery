# Consume-Mode Discovery on an IDE-Building Company

## Problem/Feature Description

Run consume-mode discovery on a company that builds a developer IDE / agent-coding tool. Public signals: their engineering blog repeatedly describes engineers using their own IDE; they ship a public "team kit" repo with internal team workflows; their hiring posts for senior eng explicitly mention `.cursor/rules`, `AGENTS.md`, and `mcp.json` as daily-life artifacts.

The company also has an SDK and a public API for the IDE — but those are for external builders, not for the company's own engineers.

Produce a discovery.json conforming to the consume-mode v3 contract.
